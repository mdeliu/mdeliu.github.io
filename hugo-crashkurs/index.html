Hugo Crashkurs

01.05.2020

Unter Windows 10 kann der statische Seitengenerator Hugo über den Chocolatey Packetmanager installiert werden und ist meiner Meinung nach ohnehin ein erstklassiger Paketmanager abseits der *nix Welt.

Dazu einfach Powershell mit Adminrechten (wichtig) starten und über folgenden Befehl Chocolatey installieren:

Set-ExecutionPolicy Bypass -Scope Process -Force; [System.Net.ServicePointManager]::SecurityProtocol = [System.Net.ServicePointManager]::SecurityProtocol -bor 3072; iex ((New-Object System.Net.WebClient).DownloadString('https://chocolatey.org/install.ps1'))

Anschließend wird Hugo über folgenden Befehl installiert:

choco install hugo -confirm

Unter Arch Linux ist eine Hugo Seite genauso schnell eingerichtet. Ich gehe davon aus, dass Pakete aus dem AUR über yay bezogen und installiert werden.

yay -S hugo-git

Anschließend kann man sich seine erste Seite erstellen. Dafür eine neue Hugo Seite erstellen und in das gewünschte Verzeichnis wechseln:

hugo new site Beispielseite

cd Beispielseite

Das erste Theme (hier “Ananke”) herunterladen und installieren (git muss installiert sein):

git init

git submodule add https://github.com/budparr/gohugo-theme-ananke.git themes/ananke

Alternativ kann man sich die Themes auch direkt herunterladen und im Websitepfad entpacken bzw. reinkopieren.

Die config.taml öffnen und folgende Zeile hinzufügen bzw. editieren:

nano config.taml

theme = "ananke"
  • je nach Theme sollte noch die Domain, Titel, Beschreibung und diverse Parameter in der Config festgelegt werden
  • Posts werden unter Beispielseite\content\posts als Markdown-File abgelegt
  • gewöhnliche Seiten eine Ebene drüber

Der erste Testlauf mit Live-Änderungen kann mit folgendem Befehl durchgeführt werden:

hugo server -D

Die URL wird direkt im Terminal ausgegeben. In der Regel localhost:1313, sofern kein weiterer Hugo Server läuft.

Über folgenden Befehl wird ein neuer Ordner public erzeugt:

hugo -D

Hier liegen alle statischen HTML-Seiten inkl. CSS, Themes und JavaScript-Files (sofern vorhanden). Diesen Inhalt direkt auf dem Webspace hochladen.
